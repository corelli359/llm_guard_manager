# 安全运营指标实现方案

## Context

围栏服务（LangTaste）产生的审计日志（audit.log）是深度嵌套的 JSON 结构，不便于直接分析。第一步需要用 Python 做日志加工：将嵌套字段展平为扁平 dict，按 app_id 分文件夹输出加工后的 JSONL 文件。后续再基于加工后的数据做指标计算和展示。

## 第一阶段：Python 日志加工脚本

### 展平规则

**audit.log 展平（嵌套 → 扁平）：**
- `request.*` → 提升到顶层（`app_id`, `apikey`, `input_prompt`, 各 use_* flag）
- `request.request_id` → `biz_request_id`
- `response.final_decision.score` → `score`（空对象时默认 0）
- `response.final_decision.priority` → `priority`（空对象时默认 -1）
- `response.all_decision_dict` → 展平为 `hit_tags`, `hit_words`, `decisions` 列表
- `status` → `http_status`
- `path` → `endpoint`
- `score` 映射为 `decision_label`：0→"pass", 50→"rewrite", 100→"block", 1000→"review"
- audit.log 中没有的字段（`safety`, `category`, `global_result`, `customize_result`, `final_result`, `decision_dict`, `vip_*_result`, `rewrite_result`, `exemption_set`, `exemption_distance`, `is_output`, `original_input_prompt`）→ 设为 null
- 丢弃元数据字段：`level`, `logger`, `method`, `context`

**request.log 展平（已扁平，直接映射）：**
- 大部分字段已在顶层，直接保留
- `final_decision.score` → `score`，`final_decision.priority` → `priority`
- `all_decision_dict` → 同样展平为 `hit_tags`, `hit_words`, `decisions`
- 补充 `decision_label`、`log_source`
- `http_status` 和 `client_ip` → 设为 null（request.log 中没有）

```python
def extract_decisions(all_decision_dict: dict) -> tuple:
    """从 all_decision_dict 提取 hit_tags, hit_words, decisions 列表"""
    hit_tags, hit_words, decisions = [], [], []
    for score_group in all_decision_dict.values():
        for tag, detail in score_group.items():
            words = detail.get("words", [])
            hit_tags.append(tag)
            hit_words.extend(words)
            decisions.append({"tag": tag, "decision": detail.get("decision"), "words": words})
    return hit_tags, list(set(hit_words)), decisions

LABEL_MAP = {0: "pass", 50: "rewrite", 100: "block", 1000: "review"}

def flatten_audit_log(raw: dict) -> dict:
    """展平 audit.log 的嵌套结构"""
    req = raw.get("request", {})
    resp = raw.get("response", {})
    fd = resp.get("final_decision", {})

    score = fd.get("score", 0)
```

用法：`python log_etl.py --input /path/to/audit.log --type audit --output /path/to/processed/`

### 处理边界情况

- `final_decision` 为空 `{}` → score=0, priority=-1, decision_label="pass"
- `status=500` 的错误请求 → 保留，decision_label="error"
- `all_decision_dict` 为空 `{}` → hit_tags=[], hit_words=[], decisions=[]
- 重复的 request_id → 跳过（基于已处理的 request_id 集合）

## 第二阶段：指标计算（基于加工后的 JSONL 文件）

读取加工后的 JSONL 文件，计算指标1和指标3，输出统计结果。

### 指标1：整体请求处理分布
- 安全通过率 = decision_label=="pass" 的数量 / 总数
- 意图改写率 = decision_label=="rewrite" 的数量 / 总数
- 直接阻断率 = decision_label=="block" 的数量 / 总数
- 支持按 app_id 过滤

### 指标3：风险事件分布
- 风险标签分布：统计 hit_tags 中各标签出现次数和占比
- 敏感词命中TOP榜：统计 hit_words 中各词频次，取 TOP N
- 敏感词命中率：命中过的去重敏感词数 / 全局敏感词库总数

### 指标4：处理时延
- 平均时延 = SUM(latency_ms) / 总请求数
- P50 时延 = latency_ms 排序后第 50% 位置的值
- P95 时延 = latency_ms 排序后第 95% 位置的值
- P99 时延 = latency_ms 排序后第 99% 位置的值
- 支持按 app_id 过滤、按时间段过滤
- 时延趋势：按小时/天聚合，输出时延变化曲线数据

### 指标5：规则与策略质量
- 规则命中率分布：统计 decisions 中每个 tag 的命中次数，识别零命中规则和高命中规则
- 单条规则拦截贡献度：每个 tag 的命中次数 / 总命中次数（所有 tag 命中之和）
- 敏感词拦截贡献度：每个 hit_word 的命中次数 / 总命中次数
- 零命中规则清单：对比全量规则标签集合与 hit_tags 去重集合，差集即为零命中规则

### 指标2：拦截有效性（需人工标注辅助）

拦截有效性无法纯自动化计算，需要结合人工抽样标注：

**自动化部分：**
- 从 decision_label=="block" 或 "rewrite" 的记录中随机抽样 N 条
- 从 decision_label=="pass" 的记录中随机抽样 M 条
- 输出抽样集到 CSV，供人工标注

**人工标注后计算：**
- 误报率 = 被标注为"正常"的拦截/改写记录数 / 抽样的拦截/改写总数
- 漏报率 = 被标注为"有害"的放行记录数 / 抽样的放行总数
- 捕获率 = 1 - 漏报率

**红队测试模式：**
- 输入标准攻击测试用例集（预标注为有害）
- 计算拦截成功率 = 被拦截或改写的数量 / 测试用例总数

### metrics_calculator.py 核心逻辑

```python
import json
import numpy as np
from pathlib import Path
from collections import Counter
from dataclasses import dataclass, field

@dataclass
class MetricsResult:
    """指标计算结果"""
    total_requests: int = 0
    # 指标1
    pass_count: int = 0
    rewrite_count: int = 0
    block_count: int = 0
    review_count: int = 0
    error_count: int = 0
    # 指标3
    tag_counter: Counter = field(default_factory=Counter)
    word_counter: Counter = field(default_factory=Counter)
    unique_hit_words: set = field(default_factory=set)
    # 指标4
    latencies: list = field(default_factory=list)
    # 指标5
    rule_hit_counter: Counter = field(default_factory=Counter)

def load_records(input_dir: str, app_id: str = None,
                 date_start: str = None, date_end: str = None) -> list:
    """加载 JSONL 记录，支持按 app_id 和日期范围过滤"""
    records = []
    base = Path(input_dir)
    app_dirs = [base / app_id] if app_id else base.iterdir()
    for app_dir in app_dirs:
        if not app_dir.is_dir():
            continue
        for jsonl_file in sorted(app_dir.glob("*.jsonl")):
            date_str = jsonl_file.stem  # "2026-01-19"
            if date_start and date_str < date_start:
                continue
            if date_end and date_str > date_end:
                continue
            with open(jsonl_file) as f:
                for line in f:
                    records.append(json.loads(line.strip()))
    return records

def calculate_metrics(records: list) -> MetricsResult:
    """计算全部指标"""
    m = MetricsResult()
    m.total_requests = len(records)
    label_map = {"pass": "pass_count", "rewrite": "rewrite_count",
                 "block": "block_count", "review": "review_count",
                 "error": "error_count"}

    for r in records:
        # 指标1: 请求分布
        label = r.get("decision_label", "unknown")
        attr = label_map.get(label)
        if attr:
            setattr(m, attr, getattr(m, attr) + 1)

        # 指标3: 风险事件
        for tag in r.get("hit_tags", []):
            m.tag_counter[tag] += 1
            m.rule_hit_counter[tag] += 1  # 指标5复用
        for word in r.get("hit_words", []):
            m.word_counter[word] += 1
            m.unique_hit_words.add(word)

        # 指标4: 时延
        lat = r.get("latency_ms")
        if lat is not None:
            m.latencies.append(lat)

    return m

def format_report(m: MetricsResult, total_keywords: int = None) -> dict:
    """格式化输出报告"""
    total = m.total_requests or 1
    report = {
        "summary": {
            "total_requests": m.total_requests,
            "pass_rate": round(m.pass_count / total * 100, 2),
            "rewrite_rate": round(m.rewrite_count / total * 100, 2),
            "block_rate": round(m.block_count / total * 100, 2),
            "review_rate": round(m.review_count / total * 100, 2),
            "error_rate": round(m.error_count / total * 100, 2),
        },
        "risk_distribution": {
            "tag_distribution": dict(m.tag_counter.most_common()),
            "top_hit_words": dict(m.word_counter.most_common(20)),
            "keyword_hit_rate": (
                round(len(m.unique_hit_words) / total_keywords * 100, 2)
                if total_keywords else None
            ),
        },
        "latency": {
            "avg_ms": round(np.mean(m.latencies), 2) if m.latencies else None,
            "p50_ms": round(np.percentile(m.latencies, 50), 2) if m.latencies else None,
            "p95_ms": round(np.percentile(m.latencies, 95), 2) if m.latencies else None,
            "p99_ms": round(np.percentile(m.latencies, 99), 2) if m.latencies else None,
        },
        "rule_quality": {
            "rule_hit_distribution": dict(m.rule_hit_counter.most_common()),
            "top_contributing_rules": _top_contributors(m.rule_hit_counter, 10),
            "top_contributing_words": _top_contributors(m.word_counter, 10),
        },
    }
    return report

def _top_contributors(counter: Counter, top_n: int) -> list:
    """计算贡献度 TOP N"""
    total = sum(counter.values()) or 1
    return [
        {"name": name, "count": count,
         "contribution": round(count / total * 100, 2)}
        for name, count in counter.most_common(top_n)
    ]
```

### 数据库表设计

#### metrics_snapshot（指标快照表）

存储每次计算的指标快照，支持历史趋势查询。

```sql
CREATE TABLE metrics_snapshot (
    id VARCHAR(36) PRIMARY KEY,
    app_id VARCHAR(64) DEFAULT NULL COMMENT '场景ID，NULL表示全局',
    snapshot_date DATE NOT NULL COMMENT '统计日期',
    total_requests INT NOT NULL DEFAULT 0,
    pass_count INT NOT NULL DEFAULT 0,
    rewrite_count INT NOT NULL DEFAULT 0,
    block_count INT NOT NULL DEFAULT 0,
    review_count INT NOT NULL DEFAULT 0,
    error_count INT NOT NULL DEFAULT 0,
    avg_latency_ms DECIMAL(10,2) DEFAULT NULL,
    p50_latency_ms DECIMAL(10,2) DEFAULT NULL,
    p95_latency_ms DECIMAL(10,2) DEFAULT NULL,
    p99_latency_ms DECIMAL(10,2) DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_app_date (app_id, snapshot_date)
) COMMENT='指标快照表';
```

#### metrics_tag_stats（标签命中统计表）

```sql
CREATE TABLE metrics_tag_stats (

    UNIQUE KEY uk_app_date_tag (app_id, snapshot_date, tag_code)
```

#### metrics_word_stats（敏感词命中统计表）

```sql
CREATE TABLE metrics_word_stats (
    id VARCHAR(36) PRIMARY KEY,
    app_id VARCHAR(64) DEFAULT NULL,
    snapshot_date DATE NOT NULL,
    keyword VARCHAR(256) NOT NULL COMMENT '敏感词',
    hit_count INT NOT NULL DEFAULT 0,
    contribution_pct DECIMAL(5,2) DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_app_date (app_id, snapshot_date),
    INDEX idx_hit_count (hit_count DESC)
) COMMENT='敏感词命中统计表';
```

## 第五阶段：数据同步定时任务

### Python 定时写入

`metrics_calculator.py` 增加 `--sync-db` 模式，计算完成后将结果写入 MySQL 表。

```bash
# 每日凌晨 2:00 执行，计算前一天的指标
0 2 * * * cd /path/to/backend/scripts && python metrics_calculator.py \
    --input /path/to/processed/ \
    --date yesterday \
    --sync-db \
    --db-url "mysql://root:xxx@127.0.0.1:38306/llm_safe_db"
```

### 全量 vs 增量

- 日常运行：只计算 `--date yesterday`，增量写入当天快照
- 历史补算：`--date-range 2026-01-01 2026-02-28`，批量计算并写入
- 写入策略：`INSERT ... ON DUPLICATE KEY UPDATE`，支持重跑覆盖
