# 安全运营指标
## 1.整体请求处理分布
  安全通过率：(状态为安全的请求数 / 总请求数) * 100%
  意图改写率：(状态为 重新的请求数 / 总请求数) * 100%
  直接阻断率 ：(状态为 拦截 的请求数 / 总请求数) * 100%
  支持下钻到场景维度
	
## 2.拦截有效性（抽样复核标注）
  漏报率 / 捕获率：通过定期红队测试或对放行请求（original_safe）的人工抽样审计来衡量。例如，投入1000条标准攻击测试用例，计算被成功拦截或改写的比例。
  误报率：统计被误改写（rewritten） 或误阻断（blocked） 的正常业务请求比例。可通过用户反馈通道或对高置信度正常请求的监控来发现。

## 3.风险事件分布
  统计命中的风险标签（A.1到A.5）及各小类的数量与占比。这能揭示当前主要风险来源，指导资源投入方向。
  敏感词命中TOP榜：列出命中频率最高的敏感词及其触发的风险标签，用于评估词库的有效性和精准性。
  敏感词命中率：命中过的敏感词/总敏感词数量
  
## 4.平均处理时延 & P99时延
  分别测量规则引擎、意图改写模块（LLM调用） 以及全链路的耗时

## 5.规则与策略质量：
  规则命中率分布：找出长期零命中的规则（可能已过时）和超高命中的规则（可能是过于宽泛的兜底规则），用于优化规则集。
  单条规则/敏感词的拦截贡献度：分析哪条规则或哪个敏感词贡献了最多的有效拦截，识别核心安全资产。
